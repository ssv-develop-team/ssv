# AGENTS.md

本文件约束本仓库内的 agent 协作方式。所有后续设计、文档和实现工作都应优先遵守这里的规则；若用户在会话中给出更具体要求，以用户最新要求为准。

## 语言与文档规范

1. 仓库内面向人的文档统一使用中文。
2. Markdown 文件名优先使用中文主题，例如 `2026-05-21-安全帽佩戴视频监测分析系统设计.md`。
3. 正文标题、图中节点、表格字段、说明段落均使用中文。
4. 技术名保留原文，不强行翻译，例如 `GStreamer`、`YOLO`、`Redis Streams`、`ONNX Runtime`、`TensorRT`。
5. 避免使用英文模板化章节名，例如 `Overview`、`Scope`、`Architecture`；对应使用 `背景`、`目标`、`范围`、`总体架构`、`技术栈选型`。
6. 文档中不要保留 `TODO`、`TBD`、`待补充` 等占位内容。暂不展开的内容应明确写入“非本阶段范围”。

## 设计流程

1. 进行创意设计、架构设计、功能新增或行为修改前，应先确认目标、范围和约束。
2. 设计确认期间应先探索仓库现状，再提出必要的澄清问题、候选方案和推荐方案。
3. 未经用户认可，不要直接写正式设计文档或实现代码。
4. 正式设计文档放在 `docs/specs/` 下。
5. 如果只是临时草稿，应在文档中标明草稿性质，或放在普通 `docs/` 下，避免和已确认 spec 混淆。

## 当前项目边界

当前项目主题是“安全帽佩戴场景下的 YOLO 模型结合多模态工具调用型 agent 的视频监测分析系统”。

第一阶段只关注整体设计：

1. 输出总体架构图。
2. 输出事件触发工作流图。
3. 明确技术栈选型。
4. 明确 `GStreamer` 作为实时视频分析主框架。
5. 明确 `GStreamer` 插件化工作流的职责边界。
6. 明确 `Redis` 异步队列用于从检测事件触发 agent 工作流。
7. 明确 agent 的软件层级、自研 LLM 状态机编排边界、工具调用边界、模型 provider 边界和向量数据库边界。

第一阶段暂不展开：

1. `GStreamer` 插件源码内部实现。
2. `YOLO` 模型训练流程。
3. agent 工具调用 schema 和提示词细节。
4. 前端页面设计。
5. 数据库表结构。
6. 平台级设备管理和大规模部署细节。
7. 完整 agent 框架实现和工具协议源码。

## 当前已确认的设计取向

1. 架构主线采用“实时检测优先”。
2. 第一版按“单机原型”考虑，视频源优先支持本地 `RTSP` 或视频文件。
3. `GStreamer + YOLO` 是实时检测主链路。
4. `Redis` 是实时检测链路与多模态 agent 工作流之间的异步边界。
5. agent 不进入每帧同步推理链路，只处理违规、低置信度或需要复核的事件。
6. `WVP`、`ZLMediaKit` 等平台集成只作为后续方向，不作为第一阶段主路径。

## Mermaid 图规范

1. Mermaid 图中的节点文本使用中文。
2. 技术名保留原文。
3. 架构图优先使用 `flowchart`。
4. 事件链路优先使用 `sequenceDiagram`。
5. 节点命名应体现职责边界，不写实现细节。

## 工作区纪律

1. 不删除用户已有文件，除非用户明确要求。
2. 不擅自回滚或清理与当前任务无关的变更。
3. 新增文档后应检查 `git status --short`。
4. 对已中断或被用户纠正的工作，应先说明当前状态，再继续。
